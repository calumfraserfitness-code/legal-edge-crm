How it works: Every time you save a lead, log a call, or update a task — the change writes to localStorage instantly (so the UI stays fast) and also syncs to Supabase in the background. Your VA opens the same HTML file, enters the same credentials, and sees the same data. Use ⬇ Pull to fetch latest data from Supabase on any device.
VA Access Guide

How your VA should use this CRM safely:

📂 Opening the file
Send your VA the HTML file once. They open it in Chrome or Edge, go to Settings → Supabase Backend, paste the Project URL and Anon Key you share with them, and click Save & Connect. From that point they're working on the same live database as you.
👤 Adding leads
Use the ＋ Add Lead button. Or import via CSV. Duplicate detection runs automatically on phone and email.
📋 Updating stages
Click any lead to open the detail panel → Edit Lead → update the Stage field. Or drag cards on the Pipeline Board.
📞 Logging calls
Click the 📞 button on any lead row or inside the detail panel. Select the outcome, add notes, set next follow-up date.
🔐 How your VA logs in
Once Supabase is connected, your VA's "login" is simply the same Project URL and Anon Key you use. There's no separate username or password system needed at this stage.

Step-by-step for your VA:
1. Download the CRM HTML file (you send it once — it never changes)
2. Open it in Chrome or Edge
3. Go to Settings → Supabase Backend
4. Paste the Project URL and Anon Key you share with them
5. Click Save & Connect — the app reloads and pulls live data

From that point: every lead they add, every call they log, every stage they update — appears on your screen instantly the next time you open the CRM (or hit Pull).

VA permissions: Switch their Role to VA in the User Profile section above. This hides revenue figures, the Reports tab, and Settings — they only see what they need to work.
